# New Project Checklist

- [ ] Independently deployable applications are separated.
- [ ] Each application is organized by business feature.
- [ ] Project-controlled names use complete descriptive words.
- [ ] Standard technical initialisms are used only when conventional.
- [ ] Shared packages have demonstrated multi-consumer ownership.
- [ ] Application startup contains wiring, not business rules.
- [ ] No empty ceremonial architecture folders were created.
- [ ] Formatting, linting, typing, building, and testing commands are documented.
- [ ] Environment secrets are excluded from version control.
- [ ] The repository tree reveals where major capabilities live.
