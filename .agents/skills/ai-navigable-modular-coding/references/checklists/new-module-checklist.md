# New Module Checklist

- [ ] The module is named after one business capability.
- [ ] `MODULE.md` defines ownership and public operations.
- [ ] Public interface and public contracts are explicit.
- [ ] External callers cannot require internal imports.
- [ ] Significant operations use named request and result contracts.
- [ ] Business decisions are separate from transport and persistence.
- [ ] Side effects are isolated behind descriptive boundaries.
- [ ] Every file has one clear primary responsibility.
- [ ] No folder exists without a current responsibility.
- [ ] Names contain complete words and business subjects.
- [ ] Public behavior and important rules have tests.
- [ ] The module can be located and understood from the tree and `MODULE.md`.
