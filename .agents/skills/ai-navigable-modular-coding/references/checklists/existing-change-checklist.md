# Existing Change Checklist

- [ ] Repository and module instructions were read.
- [ ] The function registry was read or the tree was inspected before broad file reading.
- [ ] The responsible application and module were identified.
- [ ] `MODULE.md`, public interface, relevant contracts, implementation, and tests were inspected.
- [ ] Exact file locations of target symbols were queried and confirmed via \`project-index query\` before opening files.
- [ ] The smallest expected change set was identified.
- [ ] Public contracts remain unchanged unless intentionally modified.
- [ ] Unrelated files, formatting, dependencies, and names remain untouched.
- [ ] New names use complete descriptive words.
- [ ] Relevant regression or behavior tests were added or updated.
- [ ] Confirmed callers and dependencies were checked.
- [ ] Documentation was updated only when structural information changed.
